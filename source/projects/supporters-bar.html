Supporter's Bar

User Story

If you spend enough time in an online community for a soccer team. Someone eventually will ask a question along the lines of "Are there any good bars for watching the game in #{LOCATION}?" Supporter's Bar intends to answer that question, a little bit more througouhly. Utilizing some data scraped from an existing list, I built a backend, and front end for soccer fans.

Supporter's Bar was built to answer this question, primarily for myself when I travel, but with enough data, it could become the go to source for soccer fans.

Technology Used

  • Ruby on Rails
  • PostgreSQL
  • React
  • MapBox and LeafletJS
  • Google Places API
  • Webpack with CoffeeScript

Components

Interacive Map

Much like Maptribution, Supporter's Bar utilized Leaflet, MapBox, and React to implement the map. The cheif difference between the two, is where the backend is focused. Using PostGIS, and Ruby's Geocoder gem, we search the database for locations that meet our criteria near our starting Lat/Long.

Google Places API

While I started out with a dataset of bars already. I thought that "You can watch the game at #{BAR}" was a pretty boring answer. Using the Google Places API, I'm able to pull some extra data. Like contact information, price range, and overall rating on Google.
